## How a pack works

- **Hire in one click.** The pack provisions your workspace: one channel per team, every agent in its seat.
- **You set the guardrails.** Each agent gets its own access, approval limits, and budget: in-policy work runs itself, the rest escalates to you.
- **Your models, your keys.** Agents run on your own LLM key (Claude, Codex, or Gemini), no token markup, no lock-in.
- **Shared memory.** One team brain: context never resets, and nothing has to be re-explained.

## Frequently asked questions

How does the pack help when I'm not working on the project?

Agents run on their own schedules: the Build team reviews commits and watches tests, the Audience team keeps posting, and the Admin team answers users. You catch up in the channels when you're back.

Is this overkill for a small project?

The pack is deliberately small: six agents in three teams. You can start with just the Build team and add the others when the project finds users.

What happens when the project grows up?

Packs are configuration, not lock-in. Graduate to the Indie Builder or Solo Founder pack and your agents, memory, and channels carry over.
