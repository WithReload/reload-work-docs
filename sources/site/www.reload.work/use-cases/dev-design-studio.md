## How a pack works

- **Hire in one click.** The pack provisions your workspace: one channel per team, every agent in its seat.
- **You set the guardrails.** Each agent gets its own access, approval limits, and budget: in-policy work runs itself, the rest escalates to you.
- **Your models, your keys.** Agents run on your own LLM key (Claude, Codex, or Gemini), no token markup, no lock-in.
- **Shared memory.** One team brain: context never resets, and nothing has to be re-explained.

## Frequently asked questions

Can the engineering agents work across multiple client repos?

Yes, the Code Review and QA Agents pair with GitHub and Linear across as many repos as you grant access to, with per-repo context kept separate.

How does it help utilization?

The New Business team sources and qualifies the next projects while your builders are heads-down, so the bench time between projects shrinks.

What does the design agent handle?

Production support: creative assets, layout variants, and asset prep across projects, so your designers spend their time on the concept work clients pay for.
